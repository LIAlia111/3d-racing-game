# 无限高速公路 3D 赛车

单文件网页版 3D 赛车游戏，用 [Three.js](https://threejs.org/)（通过 CDN 引入）渲染，无需构建工具、无需安装任何依赖。

## 如何运行

直接双击 `index.html`，用浏览器打开即可（推荐 Chrome / Edge）。

> 首次运行需要联网加载一次 Three.js 渲染库（约 600KB，来自 cdnjs CDN），之后浏览器会缓存。没有网络会显示加载失败提示。

## 操作说明

| 按键 | 功能 |
|---|---|
| `W` / `↑` | 加速 |
| `S` / `↓` | 刹车 / 倒退 |
| `A` / `D` 或 `←` / `→` | 左右转向（自由横移，非固定车道） |
| `Shift` 或 `空格` | 氮气加速（消耗氮气槽，松开后自动恢复） |
| `C` 或 `V` | 切换第一人称 / 第三人称视角 |
| `R` | 撞车后重新开始 |

## 功能实现对照

| 需求 | 实现方式 |
|---|---|
| 1. 第一 / 第三人称自由切换 | `updateCamera()`，按 `C`/`V` 切换 `viewMode`，两种视角各自计算摄像机位置与朝向 |
| 2. 无限高速公路 | 道路由 14 段（`CHUNK_LENGTH=60`）拼接循环回收：段落随速度前移，移出视野（`RECYCLE_Z`）后瞬移回队尾，配合雾效隐藏拼接痕迹，实现视觉上无限延伸 |
| 3. AI 车辆 | 7 辆 AI 车各自拥有独立巡航速度和车道，超出视野后在远处随机车道重新生成，形成持续车流 |
| 4. 氮气加速 | 按住 `Shift`/`空格`，速度上限与加速度提升，氮气槽消耗/恢复，触发尾焰特效与全屏光晕、摄像机 FOV 变化 |
| 5. 速度仪表盘 | 右下角 Canvas 绘制的模拟仪表盘（弧形刻度 + 指针 + 数字读数） |
| 6. 计时系统 | 左上角显示存活时间（mm:ss）与累计行驶距离（作为得分） |
| 7. 网页电脑端键盘控制 | 纯键盘输入，不依赖鼠标/触屏 |
| 8. 单文件运行 | 全部 HTML + CSS + JS 写在一个 `index.html` 内，双击即可运行 |
| 9. 完整代码 | 无省略、无 TODO，可直接运行的完整实现 |

## 代码结构（`index.html` 内 `<script>`）

- **常量配置**：车道宽度、道路分段长度、最高速度、氮气参数、AI 车速度范围等
- **场景构建**：`createRoadChunk()` 生成一段道路（含车道线、护栏、路灯），`createCarMesh()` 生成通用车辆模型（玩家与 AI 共用）
- **AI 车辆管理**：`spawnAiCars()` / `resetAiCar()` 负责生成与循环重生
- **每帧更新**：
  - `updatePlayer()` — 处理加速/刹车/转向/氮气等玩家输入
  - `updateWorldScroll()` — 道路与 AI 车的相对位移、循环回收、碰撞检测
  - `updateCamera()` / `updateIdleCamera()` — 摄像机跟随与待机镜头
  - `updateHud()` / `drawGauge()` — HUD 数据刷新与仪表盘绘制
- **游戏状态机**：`ready`（开始画面）→ `playing`（行驶中）→ `crashed`（撞车结算），`startGame()` / `triggerCrash()` / `resetState()` 负责状态切换
- **主循环**：`animate()` 基于 `requestAnimationFrame` + `THREE.Clock` 的帧间隔驱动全部逻辑

### 无限赛道的核心技巧

游戏不会真的无限延伸坐标（避免浮点精度问题），而是让**玩家车辆的 Z 坐标始终固定在 0**，反过来让道路、AI 车整体按玩家速度向摄像机方向平移；平移出视野之后再把对象"传送"回队尾远处。传送发生的位置刻意选在摄像机可视范围之外（`RECYCLE_Z` 留有安全余量），加上远处的雾效遮挡，因此玩家看不到任何拼接跳变。

## 已知限制 / 可拓展方向

- AI 车之间没有互相避让逻辑，极小概率会在同一车道重叠生成（纯视觉瑕疵，不影响可玩性）
- 没有加入音效/ 手柄支持 / 移动端触屏控制（题目要求为电脑端键盘控制）
- 计分目前用"行驶距离"代替，如需排行榜可自行加 `localStorage` 记录最佳成绩
